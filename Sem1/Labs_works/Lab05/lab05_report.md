## Задание

В соответствии с индивидуальным вариантом необходимо выполнить задание №27.

Дана матрица A размером N × M. Требуется:

1. Вывести сумму всех элементов исходной матрицы.  
2. Преобразовать матрицу, интерпретируя её элементы как линейную последовательность: заменить первый элемент разностью первого и второго, второй — разностью второго и третьего, … последний элемент остаётся неизменным.  
3. Вывести сумму элементов преобразованной матрицы.

Пользователь вручную вводит размеры матрицы и её элементы. Требуется обеспечить защиту от некорректного ввода.

---

## Разбор решения

1. Ввод размеров матрицы.  
Пользователь вводит значения N и M. Реализована защита от некорректного ввода: при ошибке поток очищается и ввод повторяется.

2. Динамическое выделение памяти.  
Создаётся двумерный массив переменного размера с помощью конструкции new int*[N] и new int[M] для каждой строки. Такой подход позволяет работать с матрицами произвольной величины.

3. Безопасный ввод элементов матрицы.  
Каждый элемент вводится вручную. Если пользователь вводит некорректные данные (буквы, символы), программа сообщает об ошибке, очищает поток и повторяет ввод.

4. Вывод исходной матрицы.  
Матрица печатается в табличной форме, что позволяет визуально проверить корректность данных.

5. Вычисление суммы исходной матрицы.  
В цикле суммируются все элементы. Используется отдельная функция OriginalSum.

6. Преобразование матрицы по правилу линейных разностей.  
Матрица рассматривается как линейная последовательность длиной N*M. Для каждого элемента k (кроме последнего) выполняется операция: A[k] = A[k] - A[k+1]. Последний элемент остаётся неизменным. Индексация реализуется через вычисление строки и столбца: i = k / M, j = k % M.

7. Вывод преобразованной матрицы.  
После преобразования матрица выводится построчно.

8. Вычисление суммы преобразованной матрицы.  
Используется функция NewSum, проходящая по всем элементам.

9. Освобождение памяти.  
Каждая строка удаляется оператором delete[], затем удаляется массив указателей. Это предотвращает утечки памяти.

# Вариативная часть: Изменение цветовой схемы консоли на основе RGB

Программа расширена возможностью изменения цветовой схемы консоли в зависимости от RGB-значений, заданных в коде. Значения BG_R, BG_G, BG_B могут быть изменены преподавателем вручную, что позволяет тестировать различные цветовые темы.

Поскольку консоль Windows поддерживает только 16 цветов, RGB нельзя отобразить напрямую. Вместо этого реализован механизм выбора одной из трёх подготовленных цветовых тем на основе яркости RGB. Яркость вычисляется по формуле:
```cpp
L = 0.2126 * R + 0.7152 * G + 0.0722 * B
```

Используя значение яркости, выбирается фоновый цвет:

- если яркость < 80 → тёмно-синий фон;  
- если яркость < 160 → тёмно-серый фон;  
- иначе → светло-серый фон.

Цвет текста определяется автоматически:  
- для светлого фона — тёмный текст,  
- для тёмного — светлый текст.  
Это сохраняет читаемость вне зависимости от выбранного RGB.

Перед изменением цвета программа сохраняет текущие атрибуты консоли через GetConsoleScreenBufferInfo, а по завершении работы восстанавливает их командой SetConsoleTextAttribute, чтобы вернуть исходную цветовую схему.

Таким образом, вариативная часть демонстрирует работу с Windows Console API, обработку RGB-параметров и реализацию адаптивной цветовой темы.
